# Fragment 三屏交互规格

本文把完整流程图、交互逻辑文档与 `orbit-fragment-interface` 前端 Demo 对齐为当前 ArkUI 实现基线。

## 1. Box

- 应用默认进入 Box，不先展示列表或分类；
- 点击盒子打开纸条输入面板；
- 碎片内容必填，Tag 可选，提醒偏好有默认值，可见范围必须明确选择；
- 完成存档后返回 Box，并显示短暂的“等待变化”反馈；
- 保存满 12 小时后，应用按通勤、睡前、周末时间窗口筛选最早的一张碎片作为低频回响；
- 点击回响卡片会回到该碎片；应用外的系统卡片和通知调度留待 Form Kit 与 Notification Kit 接入。

## 2. 个人碎片轨道与脑洞图

- 上下滑动浏览旧碎片；
- 可按关键词、Tag 和碎片类型主动搜索；
- 选择一张碎片后可以缩小为个人关系图；
- 点击个人旧碎片节点会回到对应碎片；
- 新写或从社区收存的碎片会成为脑洞图节点，并连接到操作前的中心碎片；
- 点击社区节点进入社区延申树；
- 点击“继续写”会基于当前方向打开新纸条。

## 3. 社区延申树

- 根节点必须是明确选择匿名共享的个人碎片；
- 当前碎片为私密时，进入社区会自动切换到第一张公开碎片；若没有公开碎片则停留在个人页；
- 第一层包含相似经验、相反经验和行动方向；
- 点击第一层节点后才展开下一层，避免一次性灌入大量信息；
- 选中社区节点后可“收进 Box”或“继续延申”；
- 收存的社区内容会成为一条新的私密个人碎片；
- 社区起点选择器只展示允许匿名共享的碎片；
- 社区页不提供点赞、排行、关注或无限推荐流。

## 4. 尚未接入的系统能力

- ArkData 本地数据库与跨设备同步；
- Form Kit 桌面回响卡片；
- Notification Kit 通知调度；
- 车机唤醒语音输入；
- 图片、录音和视频碎片；
- 真实账号、社区服务与内容安全审核；
- 端侧向量检索和真实 AI 关联。
