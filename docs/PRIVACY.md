# 隐私设计

## 默认与授权

- 新碎片默认 `PRIVATE`；
- 用户必须明确选择“匿名加入社区”或“仅自己可见”才能保存；
- 只有 `COMMUNITY` 且未归档的碎片可作为社区根；
- `SPECIFIC_PEOPLE` 已进入数据模型，但当前没有联系人选择器，因此 UI 不会伪装为已实现；
- 撤回共享后状态为 `REVOKED`，立即从社区入口过滤。

## 共享授权审计

- `fragments.visibility` 是当前可见性，`sharing_consents` 是不可覆盖的历史事件；
- 每次可见性变化都在同一数据库事务中关闭旧 active 事件并写入新事件；
- 新事件的 `consented_at` 使用实际操作时间，不借用碎片创建时间；
- `PRIVATE`、`COMMUNITY`、`SPECIFIC_PEOPLE`、`REVOKED` 都是明确状态；
- 同一碎片最多一条 active 事件；永久删除才会清除该碎片的全部授权历史。

## 幕后整理与 AI

- AI 授权与社区共享授权分离，并且必须对每张碎片单独选择；
- 未授权碎片不会发送到 AI 网关；
- 授权后只发送当前碎片内容、标签、提醒偏好、碎片 ID 和语言，不批量上传历史库；
- DeepSeek 与 DashScope 密钥只保存在 Cloudflare Worker Secret 中，不进入源码、数据库或安装包；
- 网关 URL 可以持久化，访问口令只保留在本次运行内存；
- 1024 维向量与分析结果保存在本机，历史碎片相似度在设备上计算；
- Worker 日志不记录碎片正文、向量或任何密钥。

## 社区收存

社区内容收存后成为新的私密个人副本，不继承公开状态。副本记录来源社区节点，独立进入个人关系图。社区节点身份由根碎片、父节点、分支类型、规范化内容和来源身份确定，展开顺序或重启不会改变。来源 ID 唯一，重复操作不会再复制一份；永久删除副本会同步移除收存索引，从而允许用户以后重新收存。

## 生命周期

- 归档是可恢复的本地状态，归档碎片不参与回响；
- 永久删除需要确认，并在同一事务中清理关系、回响、共享授权、AI 授权、AI 分析、向量和社区保存索引；
- 未选择 AI 的内容只存本机 RDB；选择 AI 后，提供商会按各自服务条款处理本张请求，应用不会把这些数据用于自有模型训练。

## 当前缺口

当前没有数据库加密密钥管理、账号、Supabase、真实社区服务、数据导出、提供商侧远程删除、举报审核或跨设备同步。Notification Kit 与 Form Kit 也尚未接入。正式公开测试前必须补充隐私政策、提供商数据保留说明、网关口令轮换与吊销、成本告警和滥用监控。
