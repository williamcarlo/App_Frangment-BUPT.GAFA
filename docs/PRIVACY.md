# 隐私设计

## 默认与授权

- 新碎片默认 `PRIVATE`；
- 用户必须明确选择“匿名加入社区”或“仅自己可见”才能保存；
- 只有 `COMMUNITY` 且未归档的碎片可作为社区根；
- `SPECIFIC_PEOPLE` 已进入数据模型，但当前没有联系人选择器，因此 UI 不会伪装为已实现；
- 撤回共享后状态为 `REVOKED`，立即从社区入口过滤。

## 社区收存

社区内容收存后成为新的私密个人副本，不继承公开状态。副本记录来源社区节点，独立进入个人关系图。来源 ID 唯一，重复操作不会再复制一份。

## 生命周期

- 归档是可恢复的本地状态，归档碎片不参与回响；
- 永久删除需要确认，并在同一事务中清理关系、回响、授权和社区保存索引；
- 当前数据只存本机 RDB，没有上传或模型训练。

## 当前缺口

当前没有数据库加密密钥管理、账号、服务端、数据导出、远程删除、举报审核或跨设备同步。接入真实服务前必须补充传输加密、最小化日志、保留期限、注销/导出和服务端幂等策略。
