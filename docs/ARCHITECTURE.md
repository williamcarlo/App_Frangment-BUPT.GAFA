# Fragment 基础架构

## 1. 产品主线

```text
碎片输入
  ↓
提醒偏好（通勤 / 睡前 / 周末 / 随机）
  ↓
未来回响
  ↓
个人脑洞网络
  ↓
社区碎片关系图谱
  ↓
自我 / 他人 / 方向延申
```

## 2. 分层设计

| 层级 | 当前职责 | 后续替换方向 |
| --- | --- | --- |
| UI | ArkUI 页面、捕捉面板、碎片卡片、延申网络 | 多设备响应式布局与系统卡片 |
| Model | FragmentItem、ReminderPreference、ExtensionBundle | 增加媒体、地点、上下文与授权字段 |
| Repository | 内存中的增删查 | ArkData Preferences / RDB / 跨设备同步 |
| RelationService | 标签重合与规则化方向建议 | 端侧 embedding、向量数据库与可解释关系评分 |
| ReminderService | 下一阶段新增 | Notification Kit、Form Kit 与用户授权的后台调度 |
| CommunityService | 当前使用模拟公开碎片 | 登录、隐私授权、审核与真实社区图谱服务 |

## 3. 延申网络

网络页围绕中心碎片展开三层信息：

- **自我延申**：内圈展示用户历史中相似或相反的旧碎片；
- **他人延申**：外圈展示社区中可公开连接的碎片节点；
- **方向延申**：右侧生成问题、灵感分叉与最小行动建议。

当前版本使用标签重合和规则模板，目的是先稳定验证交互。AI 后续只负责关联建议与问题生成，不擅自替用户下结论。

## 4. 隐私原则

- 默认仅自己可见；
- 公开必须由用户逐条选择；
- 私人碎片不进入社区检索与模型训练；
- 延申建议应说明连接原因；
- 社区不设计公开点赞数、粉丝排行与无限信息流。
