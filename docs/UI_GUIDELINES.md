# Fragment / BOX ArkUI UI 规范

> 适用范围：DevEco Studio 打开的 HarmonyOS Stage 工程 `App_Present-BUPT.GAFA`，核心页面位于 `entry/src/main/ets/pages/Index.ets`。  
> 目标：后续继续改 BOX 首页、个人碎片页、个人脑洞网络、社区树状图、输入 Sheet 时，保持统一的低饱和、纸面感、轻交互风格。

---

## 1. 设计关键词

### 1.1 产品气质

Fragment / BOX 不是效率工具的“任务面板”，也不是 AI 概念演示页，而是一个接住零散想法的轻量容器。整体视觉应保持：

- **低压力**：不要强刺激、不要高对比彩色提醒、不要密集信息流。
- **纸面感**：背景像米白纸面，卡片像轻薄纸片或柔和面板。
- **轻漂浮**：盒子、卡片、节点允许有少量阴影和透明层次，但不能厚重拟物。
- **关系感**：个人页和社区页要体现“碎片之间被慢慢连接”，不是社交媒体 feed。
- **无 AI 前台感**：智能能力只在幕后辅助整理，主界面不把 AI 当作卖点，不使用模型感、科技感或炫技式视觉。

### 1.2 禁止方向

后续 UI 调整时避免：

- 大面积紫色、蓝紫渐变、荧光色、高饱和品牌色；
- 常规社交 App 的点赞榜、瀑布流强卡片、红点轰炸；
- Material 式强阴影、强投影、强层级；
- 过多 icon、emoji、装饰图形；
- 节点关系图做成复杂知识图谱、区块链网络或 AI 工作流示意图；
- 过度圆润的糖果风按钮；
- 在主界面反复出现“AI 匹配 / 智能生成 / 模型处理”等前台文案；


### 1.3 前台文案原则

产品可以有幕后整理能力，但界面语言要像真实记录产品：

- 用“关联依据”替代“AI 匹配”；
- 用“继续展开”替代“智能生成”；
- 用“另一种视角 / 可借鉴线索”替代“模型判断”；
- 不向用户解释处理过程，只呈现清楚、可理解、可操作的结果。

## 3. 全局 Design Tokens

当前 ArkUI 的视觉 token 在 `Index.ets` 的 `BoxStyle` 中维护。新增组件必须优先复用这些 token，不要在组件内随意新增高饱和颜色。

### 3.1 颜色

| Token | 当前值 | 用途 | 使用规则 |
| --- | --- | --- | --- |
| `PAGE_BG` | `#F6F5F0` | 页面主背景 | 所有一级页面背景统一使用 |
| `SURFACE` | `#FFFFFF` | 主表面 | Sheet、主卡片、选中节点 |
| `CARD_BG` | `rgba(255,255,255,0.82)` | 半透明卡片 | 信息卡、浮层卡 |
| `SOFT_PANEL` | `rgba(239,240,235,0.88)` | 柔和面板 | 社区树、个人图谱容器 |
| `INPUT_BG` | `#F3F3F0` | 输入控件背景 | TextArea、TextInput |
| `BUTTON_BG` | `#262826` | 主按钮 | 只用于主行动按钮 |
| `TEXT` | `#252725` | 主文字 | 标题、当前高亮内容 |
| `TEXT_SOFT` | `#6F746E` | 次级文字 | 正文、描述 |
| `TEXT_MUTED` | `#A1A49E` | 弱文字 | 辅助说明、日期、状态 |
| `TEXT_FAINT` | `#DADBD6` | 极弱文字 | 非激活大编号 |
| `HAIRLINE` | `rgba(55,57,55,0.08)` | 细边框 | 卡片/胶囊边框 |
| `HAIRLINE_SOFT` | `rgba(55,57,55,0.05)` | 极弱边框 | 输入框、轻容器 |
| `SHADOW` | `rgba(44,47,45,0.12)` | 标准阴影 | 关键卡片 |
| `SHADOW_SOFT` | `rgba(44,47,45,0.08)` | 轻阴影 | 轻浮层、小节点 |
| `SELECTED_SOFT` | `#E4E6E0` | 选中背景 | 替代原紫色，不要再使用紫色选中 |
| `SELECTED_TEXT` | `#4E544E` | 选中文字 | 低饱和灰绿 |
| `GRAPH_ACTIVE` | `rgba(64,73,67,0.88)` | 图谱激活线 | 个人脑洞网络高亮线 |
| `GRAPH_IDLE` | `rgba(139,143,137,0.22)` | 图谱默认线 | 个人脑洞网络弱连接 |

### 3.2 色彩原则

1. **80% 米白 / 灰白，15% 灰绿，5% 情绪色点缀。**
2. 选中态必须使用低饱和灰色，不使用紫色：
   - 推荐：`#E4E6E0`、`rgba(226,228,222,0.96)`、`#4E544E`
   - 禁止：`#EAE6FA`、`#5B5476`、高饱和紫色、蓝紫渐变
3. 情绪色只允许用于小节点、小侧条、小圆点，不要成为背景主色：
   - memory：低饱和蓝灰 `#8AA9AE`
   - idea：低饱和玫瑰灰 `#B695A5`
   - self/green：低饱和绿灰 `#91A883`
   - slate：中性灰 `#A5A8A5`

## 4. 圆角、阴影、尺寸

### 4.1 圆角

| 类型 | 推荐值 | 说明 |
| --- | ---: | --- |
| 大卡片 / 页面容器 | `28-30` | 与 `RADIUS_CARD` / `RADIUS_PANEL` 对齐 |
| 底部 Sheet 顶角 | `30` | 只保留上左/上右圆角 |
| 胶囊按钮 | 高度 / 2 | 例如 44 高度对应 22 圆角 |
| 社区树节点 | `18` | 不要过圆，保持纸片感 |
| Toast | `20` | 小而轻 |

### 4.2 阴影

阴影只用于“轻轻浮起”，不能像实体卡片一样厚重。

推荐：

```ts
.shadow({ radius: 20, color: 'rgba(44, 46, 45, 0.08)', offsetX: 0, offsetY: 9 })
```

主按钮可稍重：

```ts
.shadow({ radius: 20, color: 'rgba(38, 40, 38, 0.16)', offsetX: 0, offsetY: 10 })
```

避免：

- 黑色 20% 以上大阴影；
- 多层阴影叠加；
- 过强浮雕或玻璃拟态。

## 5. 字体层级

### 5.1 字号体系

| 层级 | 字号 | 字重 | 用途 |
| --- | ---: | --- | --- |
| 页面主标题 | `28-30` | Bold | 个人碎片集合、社区发散 |
| 功能标题 | `24` | Medium/Bold | BOX 首页“放入碎片”、脑洞关系网 |
| 卡片标题 | `17-21` | Bold | 当前碎片标题、社区节点详情 |
| 正文 | `13-17` | Regular/Medium | 碎片正文、说明文字 |
| 辅助说明 | `11-13` | Regular/Medium | 日期、共享状态、提醒偏好 |
| 大编号 | `56-72` | Bold | 个人碎片页编号 |

### 5.2 个人碎片页文字规则

个人碎片页的高亮必须主要靠**颜色和字重**，不要通过大幅改变高度造成滚动 index 错乱。

当前建议：

- 当前项编号：`72`，颜色 `TEXT`
- 非当前项编号：`56`，颜色 `TEXT_FAINT`
- 当前项标题：`20`，颜色 `TEXT`
- 非当前项标题：`16`，颜色 `#969A94`
- 当前项正文：`16`，行高 `27`
- 非当前项正文：`15`，行高 `24`
- 卡片上下 padding 固定，不随 active 状态大幅变化。

## 6. 页面规范

## 6.1 BOX 首页

### 目标

让用户以最低压力“把一个碎片放进去”。首页不是内容展示页，而是入口和仪式感页面。

### 布局

- 顶部左侧保留 `BOX` BrandBadge。
- 主体居中：盒子插画 + 中文标题 + 英文副标题。
- 页面下方保留足够留白，避免信息拥挤。
- 背景使用 `AppBackground()` 的大圆、微颗粒、淡轨道。

### 盒子插画

- 保持低饱和灰白立体纸盒。
- 允许轻微上下浮动，幅度不超过 `10`。
- 阴影柔和，不能变成强 3D 图标。

### 交互

- 点击盒子区域打开输入 Sheet。
- 不在首页塞历史碎片、统计数字或推荐内容。

---

## 6.2 输入 Sheet

### 目标

让用户快速完成：内容、Tag、提醒偏好、共享范围。

### 结构顺序

1. 顶部：标题“放入一个碎片” + 关闭按钮；
2. TextArea：碎片内容；
3. TextInput：Tag；
4. 提醒偏好：通勤 / 睡前 / 周末；
5. 是否共享到社区：匿名共享 / 仅自己可见；
6. 主按钮：完成存档。

### 共享范围必选规则

- 用户必须明确选择共享范围。
- 未选择时主按钮文案为“选择后完成存档”。
- 点击后显示内联错误“请先选择共享范围”，并触发 Toast。
- 选择后错误清除，按钮恢复“完成存档”。

### 选中态

所有 pill 选中态使用低饱和灰色：

```ts
.backgroundColor(this.isShareChoiceSelected(value) ? 'rgba(226, 228, 222, 0.96)' : '#F1F1EE')
.border({ width: 1, color: this.isShareChoiceSelected(value) ? 'rgba(68, 72, 67, 0.16)' : 'rgba(55, 57, 55, 0.04)' })
```

不要再使用紫色选中背景。

---

## 6.3 个人碎片集合页

### 目标

这是“旧碎片被重新看见”的页面。用户通过上下滑动，让某条碎片进入视觉中心并高亮。

### 布局

- 顶部：标题“个人碎片集合” + 副标题 + “查看关系”按钮。
- 中部：垂直 List。
- 底部：提示胶囊“上下滑动，居中碎片自动高亮”。

### 高亮规则

- 视觉中心的碎片必须变为黑色高亮。
- `01 / 02 / 03 / 04 / 05 / 06` 每一条滑到中心时都必须高亮。
- 高亮逻辑优先使用 ArkUI `List.onScrollIndex` 的 `centerIndex`。
- 顶部 spacer 是第 0 项，真实碎片从 List item 1 开始，所以转换时要 `activeIndex = centerIndex - 1`。
- 如果 runtime 不提供 `centerIndex`，使用 `firstIndex / lastIndex` 中点兜底。

### 禁止

- 不要只用 `firstIndex` 判断高亮，否则高亮会偏顶部；
- 不要只用首尾 index 粗略分段，否则可能只命中 01 / 04 / 06；
- 不要让 active 项高度变化过大，否则滚动位置和 index 会互相影响。

---

## 6.4 个人脑洞网络页

### 目标

展示“我的碎片之间如何互相连接”。这是个人关系图，不是社区树状图。

### 视觉

- 使用 `Canvas` 绘制柔和曲线和小圆节点。
- 节点大小保持一致，激活态只改变颜色、透明度、描边，不改变节点尺寸。
- 标签短小，不超过 4-6 个汉字。

### 交互

- 点击节点后：
  - 激活相关边；
  - 如果节点对应碎片，更新 `activeIndex`；
  - 保持页面仍在图谱模式。

### 禁止

- 不要做成复杂力导向图；
- 不要出现高饱和发光线；
- 不要让节点大小代表重要性，当前版本应保持平等。

---

## 6.5 社区树状图页

### 目标

社区页表达“从我的一个碎片出发，看见别人共享的相似、相反或行动经验”。视觉应该是**纵向树状发散**，不是个人脑洞网络，也不是环绕轨道图。

### 参考视觉

优先参考：

```text
community_expanded_scaled.jpeg
community_page_v2.jpeg
```

### 布局

1. 顶部标题：碎片社区发散；
2. 说明文字：选择自己的碎片，从它开始查看别人共享内容；
3. 起点碎片选择 pill；
4. 树状图柔和面板；
5. 面板底部提示：点击末端节点，继续展开新的共享碎片。

### 树状结构

```text
当前碎片
  ├─ 相似经验
  │   └─ 城市观察
  ├─ 相反经验
  │   └─ 延伸比喻
  │       └─ 轻复盘
  └─ 行动方向
      └─ 关系提醒
```

### 节点规范

- 根节点：宽约 `136`，高约 `66`，白色背景，轻阴影。
- 一级节点：高约 `60`。
- 二级节点：高约 `54`。
- 三级节点：高约 `50`。
- 多节点同层时使用 `SpaceEvenly` 横向分布。
- 节点背景默认用半透明白灰：`rgba(248, 248, 244, 0.70)`。
- 激活节点使用白色背景 + 轻阴影 + 灰色边框。
- 节点不要大面积使用 blue / green / rose 背景，情绪色只可作为极轻辅助。

### 连线规范

- 使用垂直线 + 水平线组合；
- 颜色：`rgba(126, 130, 123, 0.22-0.28)`；
- 线宽：`1.2` 左右；
- 连接线要像“纸面上的细线”，不要像流程图粗箭头。

### 交互

- 点击节点：设置 `activeCommunityKey`；
- 点击末端节点：可生成新的下级分支，并展开到当前路径；
- 切换起点碎片时：重置到 root，并折叠为初始树。

### 禁止

- 不要改成个人脑洞网络那种 Canvas 节点图；
- 不要改成环形轨道图；
- 不要加头像、点赞、评论、热度；
- 不要让社区页变成无限信息流。

## 7. 导航与动效

### 7.1 页面导航

一级页面顺序固定：

```text
BOX → 个人碎片集合 → 社区树状图
```

- 左滑：进入下一页；
- 右滑：返回上一页；
- 底部 PageDots 显示当前页；
- 个人页里的脑洞网络是该页的模式切换，不是第四个一级页面。

### 7.2 动效

| 动效 | 建议值 |
| --- | --- |
| 快速反馈 | `180ms` |
| 页面/卡片状态变化 | `220ms` |
| Curve | `Curve.EaseOut` |
| BOX 浮动 | 约 `3200ms` 循环，幅度 `-4`，只保留呼吸感 |

动效原则：慢一点、轻一点、像纸片移动，不要弹跳过猛。

## 8. ArkUI 编码约束

### 8.1 单一视觉源

新增颜色、圆角、阴影前先检查 `BoxStyle`。如果是通用样式，应加到 `BoxStyle`，不要散落在组件里。

### 8.2 文本溢出

所有短标签和标题默认加：

```ts
.maxLines(1)
.textOverflow({ overflow: TextOverflow.Ellipsis })
```

正文建议：

```ts
.maxLines(2 或 3)
.textOverflow({ overflow: TextOverflow.Ellipsis })
```

### 8.3 高亮状态

高亮不要只靠颜色，也不要只靠尺寸。推荐组合：

- 主文字颜色变深；
- 字重略增强；
- 背景变为白色或低饱和灰；
- 边框透明度略增加；
- 阴影略增强。

### 8.4 不要引入 WebView

当前 demo 是纯 ArkUI 声明式界面，不使用 WebView / React Native / Vue。后续 UI 调整继续保持 ArkUI 原生实现。

## 9. 修改 UI 前检查清单

每次改 UI 前后检查：

- [ ] 是否仍然是米白 / 灰白 / 低饱和灰绿主调？
- [ ] 是否没有引入紫色选中态或高饱和渐变？
- [ ] BOX 首页是否仍然留白充分、只突出“放入碎片”？
- [ ] 输入 Sheet 的共享范围是否仍然必选？
- [ ] 个人碎片页 `01-06` 是否每一条居中时都能黑色高亮？
- [ ] 高亮项是否没有造成列表高度剧烈跳动？
- [ ] 个人脑洞网络是否仍然是 Canvas 小节点关系图？
- [ ] 社区页是否仍然是纵向树状图，而不是轨道图/网络图/feed？
- [ ] 所有文字是否设置了 maxLines 和 ellipsis？
- [ ] DevEco Studio Preview 是否能正常打开 `entry` 模块？

## 11. Step 6 品牌与动效补充

### 11.1 品牌视觉锚点

当前版本适合加强视觉识别，但加强方向应是“黑盒 / 纸面 / 墨色 / 灰绿”，而不是高饱和或科技感。

推荐做法：

- `BOX` 品牌胶囊使用深墨色底和白字，作为首屏视觉锚点；
- 背景使用温暖纸面色 `#F7F3EA`，不要回到冷白或纯灰；
- 主按钮和关键高亮使用深墨色 `#222520`；
- 选中态继续使用低饱和灰绿，不使用紫色；
- BOX 插画可以加深侧面和阴影，但不能变成强 3D 或科技图标。

### 11.2 动效去演示化

动效应像真实产品反馈，不像演示动画：

- BOX 自动浮动只保留轻微呼吸：`-4` / `3200ms`；
- Sheet 打开时背景 blur 不超过 `2`；
- 高频状态变化用 `180ms`；
- 页面/卡片状态变化用 `220ms`；
- Toast 只做轻提示，不做舞台式弹出；
- 高亮切换不要改变布局高度，优先通过颜色、边框、阴影反馈。

### 11.3 继续强化时的边界

可以继续增强：

- 纸张层次；
- 黑盒品牌识别；
- 个人碎片像档案卡的秩序感；
- 社区树像纸面分叉图的连接感。

不要增强：

- 霓虹、渐变、发光线；
- AI 助手气泡；
- 概念图式大动效；
- 信息流式社区视觉。
